# NOTE

- PoC reference implementations available in `rnd/`:
    - `rnd/server_model_1.zig` — single accept + `io.concurrent` dispatch (historical PoC)
    - `rnd/server_model_2.zig` — multiple workers with SO_REUSEPORT (historical PoC)
- Current default is **Model 2** (work-queue thread pool):
    - Accept threads push to `ConnQueue`; pool threads handle connections synchronously.
    - `workers = 0` → 2 accept threads (auto); `pool_size = 0` → `max(10, cpu_count * 2)` pool threads.
    - See `docs/concurrency.md` for full model comparison.
- **Model 1** (`workers = 1`) uses the caller's `io` directly with `io.concurrent()` per connection.
    - Use when you need an explicit `concurrent_limit` — see `examples/http_manual_concurrent.zig`.
